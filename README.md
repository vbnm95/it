# IT · IPO Trace

IPO Trace는 **공모를 통해 신규상장한 기업**을 대상으로,  
**IPO 당시 공모 정보 / 주요주주 명단 / 상장 후 주가 흐름**을 구조화해서 보여주는 웹앱 및 데이터 수집 프로젝트입니다.

---

## 1. 프로젝트 소개

IPO Trace의 핵심 목적은 다음 두 가지입니다.

1. **IPO 당시 주요주주 정보를 구조화해서 보여주기**
   - 증권발행실적보고서 기준으로 IPO 당시 주요주주 명단과 지분율을 저장
   - 프론트에서 주요주주 목록과 총 지분율을 직관적으로 확인

2. **상장 후 주가 흐름을 꾸준히 추적하기**
   - 최근 상장 기업의 일별 주가 데이터를 누적 저장
   - 공모가 대비 현재 수익률과 가격 흐름을 확인

즉, IPO Trace는  
“IPO 당시 ownership 구조와 상장 후 주가 흐름을 보여주는 서비스”** 를 목표로 합니다.

---

## 2. 추적 대상 정책

IPO Trace에는 **공모형 신규상장 기업만** 포함합니다.

### 포함 대상
- 일반적인 IPO 신규상장 기업
- KOSPI / KOSDAQ 상장 기업 중 최근 1년 내 상장 기업

### 제외 대상
- 스팩 / SPAC
- 리츠 / REIT
- ETF / ETN / ELW
- 선박투자 / 인프라투융자 / 인프라펀드
- 합병 / 합병등
- 분할 / 분할합병
- 주식교환 / 주식이전
- 재상장 / 우회상장
- 집합투자증권 / 수익증권 / 투자회사 계열

즉, 공시 제목에 `증권발행실적보고서`가 포함된다고 해서 모두 IPO로 인정하지 않고,  
**공모형 신규상장으로 볼 수 있는 케이스만 허용**합니다.

## 3. 현재 서비스 범위

### 포함 기능
- 최근 상장 기업 seed 적재
- IPO 당시 공모가 수집
- IPO 당시 주요주주 명단 / 지분율 파싱
- 상장 후 일별 주가 이력 수집
- 현재가 / 현재가 날짜 / 공모가 대비 수익률 갱신
- 추적 기간이 지난 기업 비활성화 처리

### 제외 기능
- 최신 주요주주 지분변화 자동 추적
- 최근 공시 목록 노출
- IPO 이후 주요주주 변동 비교 리포트 자동 생성

최신 주요주주 지분율은 공시 형식과 의미가 너무 다양해 완전 자동화 정확도를 보장하기 어렵기 때문에,  
현재 버전에서는 **IPO 당시 정보에 집중**합니다.

## 4. 데이터 소스

### KRX Open API

- 공모가 추출

## 5. 핵심 저장 구조

### `companies`
기업 기본 메타와 현재 스냅샷 저장
- 종목코드
- 회사명
- 시장구분
- 상장일
- DART corp_code
- 공모가
- 공모가 출처
- 현재가
- 현재가 날짜
- 공모가 대비 수익률
- 추적 시작일 / 종료일
- 활성 여부

### `price_daily`
일별 주가 이력 저장
- 날짜
- 시가 / 고가 / 저가 / 종가
- 거래량

### `shareholder_ipo_base`
IPO 당시 주요주주 기준선 저장
- 주주명
- 주식 종류
- IPO 당시 지분율
- IPO 당시 보유주식수
- 출처 공시번호 / 날짜

### `sync_runs`
배치 실행 로그 저장

### `company_archive_log`
추적 종료 기업 로그 저장

## 6. 주요 파싱 원칙

증권발행실적보고서 파싱은 다음 원칙으로 동작합니다.

- strict XML 파싱만으로 처리하지 않고 raw text 기반 보완
- section III(주요주주 관련 표), section VI(공모가 관련 표) 중심 추출
- HTML table의 rowspan / colspan 반영
- 다단 헤더 flatten
- 헤더 canonicalization
- 반복 헤더 텍스트 보정
- holder_role 은 `보통주`, `우선주` 등 **주식 종류만 저장**

즉, 표 형식이 다소 복잡한 문서도 최대한 안정적으로 읽되,  
저장되는 의미는 단순하고 일관되게 유지하는 방향입니다.

## 7. 주요 스크립트

### `seed_companies.py`
최근 1년 상장 기업을 기준으로 seed 적재
- KRX에서 상장 기업 목록 조회
- 순수 공모형 신규상장 필터 적용
- DART에서 IPO 공시 탐색
- 공모가 / IPO 주요주주 저장
- 당일 주가 스냅샷 저장

### `backfill_price_history.py`
최근 1년 가격 히스토리 백필
- `price_daily` 테이블에 과거 가격 이력 적재
- 더 최신 스냅샷이 없을 때만 `companies` 스냅샷 갱신

### `daily_update.py`
일일 운영 배치
- 신규 상장 기업 추가
- 기존 추적 기업 가격 갱신
- 추적 종료 기업 비활성화

### `rebuild_shareholders.py`
필요 시 IPO 당시 주요주주 정보 재생성

### `find_ipo_filing.py`
특정 기업의 IPO 관련 DART 공시 탐색 디버깅용

### `test_dart_document.py`
DART document.xml 파서 디버깅용

## 8. 실행 순서

### 1) DB 스키마 적용
Supabase SQL Editor에서 `schema.sql` 전체를 실행합니다.

### 2) 초기 seed
    python .\scripts\seed_companies.py --as-of 2026-04-02 --verbose

### 3) 최근 1년 가격 백필
    python .\scripts\backfill_price_history.py --as-of 2026-04-02 --verbose

### 4) 일일 운영
    python .\scripts\daily_update.py --verbose

## 9. 운영 시 주의사항

- KRX Open API의 최신 영업일 데이터는 시점에 따라 비어 있을 수 있습니다.
- 따라서 초기 적재나 재실행 시에는 **직전 정상 확인된 영업일 기준**으로 실행하는 것이 안전합니다.
- 이미 잘못 들어간 회사는 자동으로 사라지지 않으므로, 필요 시 DB에서 직접 삭제 후 재실행해야 합니다.
- `companies.current_price_date` 가 더 최신이면, 백필 스크립트는 더 과거 값으로 덮어쓰지 않도록 설계합니다.
